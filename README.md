# Event Operating System

منصة Django/DRF متعددة الجهات لإدارة الفعاليات والتسجيل والحضور والورش والشهادات والدعم والعمليات.

## التشغيل المحلي

```powershell
python manage.py migrate
python manage.py seed_demo
python manage.py collectstatic --noinput
python manage.py runserver 127.0.0.1:8000
```

الروابط المهمة:

- Public portal: `http://127.0.0.1:8000/`
- Dashboard: `http://127.0.0.1:8000/dashboard/`
- Django admin: `http://127.0.0.1:8000/admin/`
- Swagger/OpenAPI: `http://127.0.0.1:8000/api/docs/`
- JWT token: `http://127.0.0.1:8000/api/token/`

حسابات demo:

- `admin / admin12345`
- `orgadmin / admin12345`
- `checkin / admin12345`

## ما تم تنفيذه

- Django 5.2 وDRF وJWT وSwagger وCORS وWhiteNoise وCelery config.
- SQLite محليًا، ودعم PostgreSQL عبر `DATABASE_URL`.
- Multi-tenant models للجهات والفعاليات وصلاحيات أدوار واسعة.
- Public portal عربي RTL: صفحة فعالية، تسجيل، حالة التسجيل، تذكرة QR، جدول، ورش، شهادات، تحقق شهادة، دعم، شاشة عرض.
- Dashboard عربي RTL: إحصائيات، جهات، مستخدمين وأدوار، مدارس ومجموعات، فعاليات، modules، نموذج تسجيل، قاعات ومكان، جدول وجلسات، تسجيلات، check-in، ورش، متطوعين وشيفتات، ملاحظات طلاب، شهادات، broadcast، crisis، support، VIP/speakers/sponsors، تقييمات، إعلام، مالية، مخالفات، تقارير.
- Domain basis لكل الموديولات المذكورة: Volunteers, VIP, Speakers, Sponsors, Media, Surveys, Tickets/Finance, App updates, AI insights, backups, subscriptions.
- منطق عملي للخدمات: منع تكرار التسجيل، مراجعة الطلبات، check-in/check-out، منع check-in مكرر، تعارض الورش، نقاط وأوسمة، مخالفات تمنع الشهادة، إصدار PDF شهادة، QR فعلي، Excel export.
- إدارة التسجيلات تدعم استيراد Excel، قبول/رفض/انتظار جماعي، وتصدير Excel.
- ملف المشارك في الداشبورد يعرض ويعدل البيانات الشخصية والتعليمية، الحضور، الورش، النقاط، الأوسمة، الشهادات، المخالفات، الملاحظات، المستندات، الدعم، وسجل العمليات.
- التقارير تدعم تصدير Excel للمشاركين والحضور والشهادات والورش والمخالفات على مستوى كل الفعاليات أو فعالية محددة.
- إدارة الورش تدعم تسجيل حضور الورشة بالكود مع منح النقاط تلقائيًا، والـ API يحتوي actions مقابلة.
- إدارة القاعات والمكان تدعم أيام الفعالية، المسارات، القاعات، المسؤولين، والسعات.
- إدارة الجدول تدعم إنشاء الجلسات وربطها باليوم والمسار والقاعة والمتحدثين مع حالة النشر.
- إدارة المتطوعين تدعم تحويل طلبات المتطوعين إلى ملفات تشغيلية وتوزيع شيفتات.
- ملاحظات الطلاب تدعم ملاحظات داخلية أو ظاهرة للطالب مرتبطة بجلسة أو ورشة.
- النقاط والأوسمة تدعم قواعد تلقائية، منح/خصم يدوي، أوسمة تلقائية ويدوية، وLeaderboard عام وداخلي حسب المشاركين والمدارس والإدارات والمجموعات والمتطوعين.
- مركز الاتصالات يدعم Broadcast، قوالب بريد قابلة للتعديل، إرسال بريد لمشارك بقالب أو نص حر، سجلات بريد مع إعادة إرسال، وسجل Push/In-app آمن بدون بيانات حساسة.
- الدعم العام يدعم فتح طلب ومتابعة الطلب بالرقم مع عرض الردود العامة.
- الشهادات تدعم تسجيل محاولة إرسال البريد وإعادة الإرسال الآمن عند عدم وجود مزود خارجي.
- إدارة الشهادات تدعم إصدارًا فرديًا وجماعيًا، إرسالًا فرديًا وجماعيًا، إلغاء الشهادة، وإعادة إصدارها.
- التقييمات والاستبيانات لها واجهة عامة وإدارة أسئلة وردود، ويمكن جعلها شرطًا لإصدار الشهادة.
- المركز الإعلامي له صفحة عامة وإدارة مواد منشورة أو مسودات.
- المالية والتذاكر تشمل أنواع تذاكر، إصدار تذكرة لمشارك، تحديث حالة الدفع، إلغاء وإعادة إصدار QR، كوبونات خصم، فواتير وإيصالات، Refunds، مصروفات، وملخص صافي بعد الاستردادات.
- Mobile API توسع ليشمل لوحة طالب ومشرف ومتطوع بجانب التحقق من إصدار التطبيق وscan/check-in.
- APIs الإدارية تدعم pagination، البحث، الترتيب، وفلترة تلقائية للموارد الأساسية والمتولدة.
- إدارة المستخدمين والأدوار تتيح إنشاء مستخدم وربطه بجهة وفعاليات محددة.
- إدارة المدارس والإدارات والمجموعات تعمل من الداشبورد وتدعم ربط المجموعات بالفعاليات.
- إدارة المنصة تشمل إصدارات التطبيق، خطط الاشتراك، اشتراكات الجهات، وBackup Snapshot منطقي قابل للتوسعة.
- التقارير تدعم Snapshot للتقرير النهائي للفعالية وتحميل PDF.
- البريد وPush يعملان بنمط آمن عند غياب مفاتيح خارجية: يتم تسجيل المحاولة كـ mocked بدل انهيار النظام.
- بيانات demo تغطي فعالية تعليمية ومؤتمر، مشاركين، مدارس، قاعات، جلسات، ورش، شهادات، نقاط، مخالفات، دعم، SOS، VIP، متحدثين، رعاة.
- اختبارات تلقائية للمسارات الأساسية.

## ملاحظات تشغيل

- WeasyPrint مثبت، لكن على Windows قد يحتاج مكتبات native إضافية لتوليد PDF حقيقي. الكود يحتوي fallback لا يكسر التجربة ويسجل/يعرض المحتوى.
- Brevo/SendGrid وFirebase جاهزة كتوصيل إعدادات، ولا ترسل خارجيًا بدون مفاتيح.
- Redis مطلوب فقط عند تشغيل Celery خارج وضع `CELERY_TASK_ALWAYS_EAGER=true`.
- للإنتاج اضبط على الأقل: `DJANGO_DEBUG=false`، `DJANGO_SECRET_KEY`، `DJANGO_ALLOWED_HOSTS`، و`DATABASE_URL`. إعدادات HTTPS/HSTS والكوكيز الآمنة تعمل افتراضيًا عند إغلاق DEBUG ويمكن تخصيصها بمتغيرات `DJANGO_SECURE_*`، وفعّل `DJANGO_SECURE_HSTS_PRELOAD=true` فقط بعد جاهزية الدومين للـ preload.
