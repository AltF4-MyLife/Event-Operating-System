أنت تعمل كمهندس برمجيات Full-Stack Senior ومتخصص في Django وDjango REST Framework وبناء منصات SaaS لإدارة الفعاليات والمؤتمرات.

المطلوب منك بناء نظام كامل لإدارة أي فعالية أو مؤتمر أو تدريب أو معسكر أو حدث تابع لأي هيئة أو شركة أو مبادرة أو مدرسة أو إدارة تعليمية. النظام يجب أن يكون قابلًا للتشغيل كمنصة عامة يمكن استخدامها لأكثر من جهة وأكثر من فعالية، مع واجهة خارجية للجمهور وواجهة داخلية للإدارة والتحكم.

اشتغل بمنهج pursue-goal: استمر في التحليل، التصميم، التنفيذ، الاختبار، الإصلاح، وإعادة الاختبار حتى يصبح النظام مستقرًا وجاهزًا للاستخدام بدون أخطاء واضحة. لا تكتفي ببناء جزء بسيط. لا تترك وظائف وهمية أو غير مكتملة. لا تضع بيانات ثابتة بدل المنطق الحقيقي إلا في بيانات تجريبية واضحة. لا تسأل أسئلة إلا لو كان التنفيذ مستحيلًا بدون قرار، وفي هذه الحالة اختر أفضل افتراض عملي وكمّل.

لا تكتب شرحًا نظريًا فقط. المطلوب تنفيذ نظام حقيقي كامل قابل للتجربة من المتصفح ومن خلال API. اختبر كل مسار مهم بنفسك بعد التنفيذ. أي خطأ يظهر أثناء التشغيل أو الاختبار يجب إصلاحه فورًا ثم إعادة الاختبار.

اسم المنتج المقترح:
Event Operating System

الفكرة:
منصة كاملة لإدارة الفعاليات من أول إنشاء الجهة المنظمة والفعالية، مرورًا بالتسجيل والمراجعة والقبول وQR والحضور والورش والمتطوعين والمشرفين والإشعارات والأزمات والمخالفات، وصولًا إلى الشهادات والتقارير والأرشفة.

التقنية الأساسية المطلوبة:

* Django 5.2.x LTS، مع استخدام آخر تحديث آمن متاح من نفس الفرع.
* Django REST Framework.
* قاعدة بيانات SQLite للتجربة المحلية فقط.
* PostgreSQL للإنتاج.
* Gunicorn للتشغيل الإنتاجي.
* WhiteNoise لخدمة الملفات الثابتة.
* djangorestframework-simplejwt لمصادقة API عبر JWT.
* django-cors-headers للسماح بطلبات الويب والتطبيق.
* fcm-django لإشعارات Firebase Cloud Messaging.
* WeasyPrint لتوليد شهادات PDF.
* openpyxl لتصدير Excel.
* دعم Brevo وSendGrid لإرسال البريد الإلكتروني.
* إضافة Celery + Redis أو بديل مناسب للمهام الثقيلة مثل إرسال الإيميلات، توليد الشهادات، إرسال الإشعارات، إنشاء التقارير، وأي عمليات Batch.
* إضافة نظام API documentation مناسب مثل OpenAPI/Swagger.
* إضافة نظام فلترة وبحث وترتيب للـ APIs.
* إضافة اختبارات تلقائية للمنطق الأساسي.
* الالتزام بالمنطقة الزمنية Africa/Cairo.
* دعم كامل للغة العربية واتجاه RTL في الواجهات.
* كل الواجهات تكون عملية ونظيفة وسريعة ومناسبة للإدارة.

ممنوع تجاهل أي Module من المذكور. لو ميزة ضخمة جدًا، نفّذ أساسها العملي القابل للتوسعة، وضعها كخاصية قابلة للتفعيل والإلغاء من لوحة التحكم.

42. جودة التنفيذ:

* النظام يجب أن يكون منظمًا وقابلًا للتوسع.
* لا تخلط منطق الأعمال المعقد عشوائيًا.
* اجعل المنطق المهم قابلًا للاختبار.
* لا تجعل صفحات الداشبورد تعتمد على بيانات ثابتة.
* لا تتجاهل حالات الخطأ.
* لا تسمح بكسر النظام عند نقص إعداد بريد أو Firebase.
* في حالة عدم وجود إعدادات خارجية، سجّل العملية كغير مكتملة أو Mock آمن بدل انهيار النظام.
* اهتم بالأداء في الجداول الكبيرة.
* استخدم Pagination.
* استخدم فلاتر.
* استخدم رسائل واضحة.
* اجعل كل Feature قابلة للإغلاق بدون التأثير على باقي النظام.
